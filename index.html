Two storage methods. Zero headaches.

A flat key-value store for simple data, plus a full hierarchical file system for structured projects.

PUT /api/data/:key 
// Store anything
fetch('https://data2l.ink/api/data/myapp', {
  method: 'PUT',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    value: { users: 42, status: 'live' }
  })
})
GET /api/data/:key 
// Retrieve it
fetch('https://data2l.ink/api/data/myapp', {
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
.then(r => r.json())
.then(data => {
  console.log(data.value)
  // → { users: 42, status: 'live' }
})
GET /api/data 
// List all your keys
fetch('https://data2l.ink/api/data', {
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
.then(r => r.json())
.then(data => {
  console.log(data.keys)
  // → ['myapp', 'config', ...]
})
DELETE /api/data/:key 
// Remove a key
fetch('https://data2l.ink/api/data/myapp', {
  method: 'DELETE',
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
// → { key: 'myapp', deleted: true }

File System API (JJFS)

Store data as a virtual file tree — nested directories, POSIX paths, surgical in-place edits. One endpoint handles everything.

POST /api/fs/execute — write a file 
// Create or overwrite a file
fetch('https://data2l.ink/api/fs/execute', {
  method: 'POST',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    type: 'JJFS_WRITE',
    target: 'default:/notes/todo.md',
    content: '# My Notes\n- Item 1'
  })
})
// → { success: true, result: "Created: default:/notes/todo.md" }
POST /api/fs/execute — read a file or dir 
// Read a file
{ type: 'JJFS_READ', target: 'default:/notes/todo.md' }
// → { success: true, result: "# My Notes\n- Item 1" }

// Read lines 10–20 only
{ type: 'JJFS_READ', target: 'default:/notes/todo.md:10:20' }

// List a directory
{ type: 'JJFS_READ', target: 'default:/notes' }
// → { success: true, result: "todo.md\narchive/" }

// Structured browse (GET, for UIs)
GET /api/fs/browse?workspace=default&path=/notes
POST /api/fs/execute — surgical edit 
// Replace exactly one occurrence — must be unique
fetch('https://data2l.ink/api/fs/execute', {
  method: 'POST',
  headers: { 'X-API-Key': 'd2l_your_key', ... },
  body: JSON.stringify({
    type: 'JJFS_EDIT',
    target: 'default:/notes/todo.md',
    content: JSON.stringify({
      search: '- Item 1',
      replace: '- Item 1\n- Item 2'
    })
  })
})
// → { success: true, result: "Edited: default:/notes/todo.md" }
POST /api/fs/execute — move, copy, delete 
// Move a file
{ type: 'JJFS_MOVE',
  target: 'default:/notes/todo.md',
  content: '/archive/todo.md' }

// Copy a file
{ type: 'JJFS_COPY',
  target: 'default:/template.md',
  content: '/notes/new.md' }

// Delete a file or directory
{ type: 'JJFS_DELETE',
  target: 'default:/archive' }

// Manage workspaces
POST  /api/fs/workspaces   { name: 'my-project' }
GET   /api/fs/workspaces
DELETE /api/fs/workspaces/my-project

Built for speed, not complexity.

🗂️

Hierarchy makes data navigable

The flat key-value store is great for simple lookups. But as your data grows, scanning a list of keys gets painful. JJFS lets you organise data as a real file tree — default:/config/db.json, default:/users/profile.md, my-project:/src/app.js — so finding things is as natural as navigating a project folder. No more guessing key names.

⚡

In-memory speed

Your data lives in RAM for instant access. Every mutation is persisted to disk so nothing is ever lost.

🔑

API key scoped

Each key isolates your data. Create multiple keys for different projects — they never cross-contaminate.

📦

Store anything

Strings, objects, arrays, nested structures — if it's JSON, it goes in. No schemas, no migrations.

🌐

CORS ready

Call it from any frontend, any origin. Built-in CORS headers mean zero proxy headaches.

🛡️

Secure by default

🧩

Zero dependencies

Pure Node.js, no frameworks, no bloat. Runs anywhere that runs JavaScript.
