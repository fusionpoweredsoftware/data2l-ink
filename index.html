Two storage methods. One framework.

A flat key-value store for simple data, plus a full hierarchical file system for structured projects. KV keys can be made publicly accessible via URL.

PUT /api/data/:key 
// Store anything
fetch('https://data2l.ink/api/data/myapp', {
  method: 'PUT',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    value: { users: 42, status: 'live' }
  })
})
GET /api/data/:key 
// Retrieve it
fetch('https://data2l.ink/api/data/myapp', {
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
.then(r => r.json())
.then(data => {
  console.log(data.value)
  // → { users: 42, status: 'live' }
})
GET /api/data 
// List all your keys
fetch('https://data2l.ink/api/data', {
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
.then(r => r.json())
.then(data => {
  console.log(data.keys)
  // → ['myapp', 'config', ...]
})
DELETE /api/data/:key 
// Remove a key
fetch('https://data2l.ink/api/data/myapp', {
  method: 'DELETE',
  headers: {
    'X-API-Key': 'd2l_your_key'
  }
})
// → { key: 'myapp', deleted: true }
PATCH /api/data/:key 
// Make a key public (or private)
fetch('https://data2l.ink/api/data/myapp', {
  method: 'PATCH',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    visibility: 'public'
  })
})
// → { key: 'myapp', visibility: 'public' }
GET /d2l/:accountId/:key 
// Public link — share this URL openly
// Open it in any browser or fetch it from your code:
fetch('/d2l/your_account_id/myapp')
  .then(r => r.json())
  .then(data => {
    console.log(data.value)
    // → { users: 42, status: 'live' }
  })

File System API (JJFS)

Store data as a virtual file tree — nested directories, POSIX paths, surgical in-place edits. Standard REST verbs on clean URLs.

PUT /api/fs/:workspace/*path — write a file 
// Create or overwrite — body is raw file content
fetch('https://data2l.ink/api/fs/default/notes/todo.md', {
  method: 'PUT',
  headers: { 'X-API-Key': 'd2l_your_key' },
  body: '# My Notes\n- Item 1'
})
// → { success: true, result: "Created: default:/notes/todo.md" }

// curl
curl -X PUT https://data2l.ink/api/fs/default/notes/todo.md \
  -H 'X-API-Key: d2l_your_key' \
  --data '# My Notes'
GET /api/fs/:workspace/*path — read a file or dir 
// Read a file — returns raw text
fetch('https://data2l.ink/api/fs/default/notes/todo.md', {
  headers: { 'X-API-Key': 'd2l_your_key' }
})
.then(r => r.text())
.then(console.log)
// → "# My Notes\n- Item 1"

// Optional: read lines 10–20 only  →  ?start=10&end=20
PATCH /api/fs/:workspace/*path — surgical edit 
// Replace exactly one occurrence — must be unique
fetch('https://data2l.ink/api/fs/default/notes/todo.md', {
  method: 'PATCH',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    search: '- Item 1',
    replace: '- Item 1\n- Item 2'
  })
})
// → { success: true, result: "Edited: default:/notes/todo.md" }
DELETE /api/fs/:workspace/*path — delete 
// Delete a file or directory
fetch('https://data2l.ink/api/fs/default/notes/todo.md', {
  method: 'DELETE',
  headers: { 'X-API-Key': 'd2l_your_key' }
})
// → { success: true, result: "Deleted: default:/notes/todo.md" }
POST /api/fs/:workspace/*path — move 
// Move a file or directory
fetch('https://data2l.ink/api/fs/default/notes/todo.md', {
  method: 'POST',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    op: 'move',
    destination: '/archive/todo.md'
  })
})
// → { success: true, result: "Moved: default:/notes/todo.md → /archive/todo.md" }
POST /api/fs/:workspace/*path — copy 
// Copy a file or directory
fetch('https://data2l.ink/api/fs/default/template.md', {
  method: 'POST',
  headers: {
    'X-API-Key': 'd2l_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    op: 'copy',
    destination: '/notes/new.md'
  })
})
// → { success: true, result: "Created: default:/notes/new.md" }

Built for speed, not complexity.

⚡

In-memory speed

Your data lives in RAM for instant access. Every mutation is persisted to disk so nothing is ever lost.

🔑

Fine-grained permissions

Assign each API key its own privilege level — read-only, read-write, or admin. Lock down what each key can touch.

📦

Store anything

Strings, objects, arrays, nested structures — if it's JSON, it goes in. No schemas, no migrations.

🌐

CORS ready

Call it from any frontend, any origin. Built-in CORS headers mean zero proxy headaches.

🛡️

Secure by default

Passwords hashed with PBKDF2. API keys are long, random, and revocable at any time.

🧩

Zero dependencies

Just fetch. No SDK, no client library, no wrapper — plain HTTP. Works in any JavaScript environment.

🔗

Public shareable links

Mark any KV key as public and share it as a plain URL — anyone with the link can read the value.

⚗️

Built-in playground

Test every API call live from your dashboard — no client code needed. Dial in your requests before wiring them into your app.

🗂️

Hierarchy makes data navigable

JJFS organises data as a real file tree — default:/config/db.json — so finding things is as natural as navigating a project folder.
