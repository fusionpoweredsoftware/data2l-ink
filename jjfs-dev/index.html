Drop in jjfs.js, create a workspace object, and start reading & writing files. No config, no server, no install.

ES Modules Node.js Browser
example.js 
import { jjfsRead, jjfsWrite, jjfsEdit }
  from './jjfs.js';

// Your entire file system lives in one object
const wsForKey = { default: {} };

// Write a file (creates dirs automatically)
jjfsWrite(wsForKey, 'default', '/src/app.js',
  'console.log("hello jjfs!")');
// → { success: true, result: 'Created: default:/src/app.js' }

// Read it back
jjfsRead(wsForKey, 'default', '/src/app.js');
// → { success: true, result: 'console.log("hello jjfs!")' }

// Surgical edit (unique search → replace)
jjfsEdit(wsForKey, 'default', '/src/app.js',
  'hello jjfs!', 'hello world!');
// → { success: true, result: 'Edited: default:/src/app.js' }

// List a directory
jjfsRead(wsForKey, 'default', '/');
// → { success: true, result: 'src/' }
The Problem

AI models are stateless.
JJFS fixes that.

Every AI request arrives without memory of what came before. The moment you want the model to build something — a codebase, a document, a config — you hit a wall.

Common workaround The problem
Paste all files into every prompt Context window fills fast; expensive and slow
Summarize files server-side Information is lost in translation
Ask the model to output a diff Fragile; requires careful post-processing
Tool-call APIs (function calling) Locks you to one provider's specific format

The JJFS answer

Give the model a named virtual folder. Teach it a simple tagged syntax for reading and writing files inside that folder. Let it get to work — across as many turns as needed, with any model, in any environment.

Hierarchical workspaces

POSIX-style paths, nested directories, multiple named workspaces. The data model is just nested JSON — trivial to inspect.

✏️

Surgical edits

jjfsEdit replaces only the section you specify. Fails safely if the search text is missing or ambiguous — no accidental overwrites.

🔄

Auto-creates directories

Write to /a/b/c/file.txt and every intermediate directory is created automatically. No mkdir -p required.

💾

Persistence-agnostic

The library operates on plain objects. Wire it to localStorage, IndexedDB, a JSON file, or any database — your choice.

📋

Line-range reads

Read any slice of a large file with start:end line numbers. Essential for keeping large files from flooding the model's context.

🔀

Move & copy

Full move and deep-copy operations for files and entire directory trees. Consistently returns { success, result } — never throws.

API Reference

Nine functions.
Complete API surface.

Every function takes a wsForKey object — a map of workspace names to their directory trees. Mutating functions modify in place and always return { success, result }.

Function What it does
jjfsRead
(wsForKey, wsName, filePath, startLine?, endLine?)
 Read a file or list a directory. Optionally slice by line range. Returns file content or newline-separated directory listing.
jjfsWrite
(wsForKey, wsName, filePath, content)
 Create or overwrite a file. Intermediate directories are created automatically. Use for new files; prefer jjfsEdit for modifications.
jjfsEdit
(wsForKey, wsName, filePath, searchStr, replaceStr)
 Surgical search-and-replace. The search string must appear exactly once. Fails safely without modifying the file if ambiguous or missing.
jjfsDelete
(wsForKey, wsName, filePath)
 Remove a file or an entire directory tree. Returns an error if the path is not found rather than silently succeeding.
jjfsMove
(wsForKey, wsName, srcPath, destPath)
 Relocate a file or directory to a new path within the same workspace. Atomic: source is only removed after destination is written.
jjfsCopy
(wsForKey, wsName, srcPath, destPath)
 Deep-copy a file or directory tree to a new path. Uses JSON.parse(JSON.stringify(...)) to ensure a full structural clone.
jjfsNavigate
(workspace, pathStr)
 Resolve a POSIX path string to { parent, name }. Used internally by all path-based operations. Leading slash is optional.
parseTarget
(target, forRead?)
 Parse a "wsName:/path" action target string (or "wsName:/path:start:end" for reads). Returns { wsName, filePath, startLine?, endLine? }.
countFiles
(node)
 Recursively count all leaf files (string values) in a workspace tree. Useful for quota checks and progress reporting.

Import in your HTML

Use a type="module" script tag to import as an ES module.

index.html 
<script type="module">
  import { jjfsRead, jjfsWrite, jjfsEdit,
           jjfsDelete, jjfsMove, jjfsCopy }
    from './jjfs.js';

  const wsForKey = { default: {} };

  jjfsWrite(wsForKey, 'default', '/notes.md', '# My Notes');
  const { result } = jjfsRead(wsForKey, 'default', '/notes.md');
  console.log(result); // # My Notes
</script>

Persist to localStorage (optional)

Save the workspace to localStorage after every write to survive page reloads.

persistence.js 
// Load from storage
const wsForKey = JSON.parse(
  localStorage.getItem('jjfs') ?? '{}'
);
if (!wsForKey.default) wsForKey.default = {};

// Save after every mutating operation
function save() {
  localStorage.setItem('jjfs', JSON.stringify(wsForKey));
}

const r = jjfsWrite(wsForKey, 'default', '/todo.txt', 'Buy milk');
if (r.success) save();

Load, mutate, persist

Load workspace state from a JSON file on startup. Save it after each mutation.

server.mjs 
import fs from 'node:fs';
import { jjfsWrite, jjfsRead } from './jjfs.js';

const FILE = './workspaces.json';

function load() {
  try { return JSON.parse(fs.readFileSync(FILE, 'utf8')); }
  catch { return {}; }
}
function save(ws) {
  fs.writeFileSync(FILE, JSON.stringify(ws, null, 2));
}

const wsForKey = load();
if (!wsForKey.default) wsForKey.default = {};

const r = jjfsWrite(wsForKey, 'default', '/notes.md', '# Notes');
if (r.success) save(wsForKey);

Wire a POST /api/fs/execute endpoint

The standard integration: one endpoint dispatches all JJFS operations. The AI (or client) sends { type, target, content }.

handler.mjs 
import { parseTarget, jjfsRead, jjfsWrite,
         jjfsEdit, jjfsDelete, jjfsMove, jjfsCopy }
  from './jjfs.js';

async function handleExecute(body, wsForKey) {
  const { type, target, content } = body;
  const t = parseTarget(target, type === 'JJFS_READ');
  if (t.error) return { success: false, result: t.error };

  const { wsName, filePath, startLine, endLine } = t;
  switch (type) {
    case 'JJFS_READ':
      return jjfsRead(wsForKey, wsName, filePath, startLine, endLine);
    case 'JJFS_WRITE':
      return jjfsWrite(wsForKey, wsName, filePath, content);
    case 'JJFS_EDIT':
      const { search, replace } = JSON.parse(content);
      return jjfsEdit(wsForKey, wsName, filePath, search, replace);
    case 'JJFS_DELETE':
      return jjfsDelete(wsForKey, wsName, filePath);
    case 'JJFS_MOVE':
      return jjfsMove(wsForKey, wsName, filePath, content);
    case 'JJFS_COPY':
      return jjfsCopy(wsForKey, wsName, filePath, content);
    default:
      return { success: false, result: `Unknown type: ${type}` };
  }
}

  -H "Content-Type: application/json" \
  -d '{"type":"JJFS_READ","target":"default:/hello.txt"}'

# Both return: { "success": true, "result": "..." }

System Prompt Template 
You have access to JJFS — a persistent virtual file system.
Use it to create, read, and edit files across turns.

── WORKSPACES ───────────────────────────
  default    — General purpose workspace

── FILE ACTIONS (embed in your response) ──

READ:   [ACTION:JJFS_READ]ws:/path[/ACTION:JJFS_READ]
READ:   [ACTION:JJFS_READ]ws:/path:30:50[/ACTION:JJFS_READ]
WRITE:  [ACTION:JJFS_WRITE:ws:/path:content here[/ACTION]
EDIT:   [ACTION:JJFS_EDIT:ws:/path:{"search":"...","replace":"..."}[/ACTION]
DELETE: [ACTION:JJFS_DELETE:ws:/path[/ACTION]
MOVE:   [ACTION:JJFS_MOVE:ws:/old:/new[/ACTION]
COPY:   [ACTION:JJFS_COPY:ws:/src:/dest[/ACTION]

── RULES ────────────────────────────────

1. NEVER paste file content in your prose. Use WRITE/EDIT.
2. PREFER EDIT over WRITE for existing files.
3. EDIT: search text must appear exactly once.
4. READ relevant sections before EDIT.
5. All code goes in JJFS files, not in your response.
